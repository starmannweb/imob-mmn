2. Modelagem do Banco de Dados

Esquema otimizado para a árvore hierárquica do Multinível.

Tabela: users

id: uuid (PK)
full_name: text
document: text (CPF/CNPJ)
creci: text
referral_code: text (Unique)
referred_by: uuid (FK -> users.id)
plan_status: enum (active, inactive, trial)

Tabela: properties

id: uuid (PK)
owner_id: uuid (FK -> users.id)
title: text
price_sale: decimal
bedrooms, bathrooms, suites: smallint
slug: text (Unique)

Tabela: leads

id: uuid
property_id: uuid (FK)
assigned_to: uuid (FK -> users.id)
phone_whatsapp: text
status: enum (new, negotiating, won...)

3. Lógica do Algoritmo MMN

Para o cálculo das comissões, utilizamos queries recursivas no PostgreSQL para identificar a linha ascendente de ganho.

Hierarquia de Repasse:

  • Venda Direta: Cadastro em transactions vinculado ao seller_id.
  • 1ª Geração (Indireta): 4% para quem indicou o vendedor.
  • 2ª Geração (Indireta): 2% para quem indicou a 1ª geração.
  • 3ª Geração (Indireta): 1% para quem indicou a 2ª geração.

Implementação: Recomenda-se o uso de Edge Functions do Supabase para processar o pagamento do Asaas/Stripe e desencadear os inserts de crédito nas carteiras virtuais.
